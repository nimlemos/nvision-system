🚀 PRÓXIMOS PASSOS (RESUMIDO):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PASSO 1: Criar conta no GitHub
   └─ Acesse: https://github.com/join
   └─ Crie uma nova conta (leva 1 minuto)

PASSO 2: Criar repositório
   └─ https://github.com/new
   └─ Nome: "nvision-system"
   └─ Visibilidade: Public
   └─ Clique "Create repository"

PASSO 3: Enviar seu código
   No terminal/PowerShell, na pasta do projeto:
   
   git init
   git add .
   git commit -m "Projeto NVision - Deploy inicial"
   git branch -M main
   git remote add origin https://github.com/SEU_USUARIO/nvision-system.git
   git push -u origin main

PASSO 4: Fazer deploy no Render
   └─ https://render.com
   └─ Sign up com GitHub
   └─ "New" → "Web Service"
   └─ Selecione o repositório nvision-system
   └─ Clique "Create Web Service"
   └─ Aguarde 3-5 minutos

PASSO 5: Acessar seu app
   Render fornecerá URL: https://nvision-system.onrender.com
   Login: admin / admin123


⚠️ IMPORTANTE SABER:

✓ GRÁTIS: Render oferece tier gratuito
✓ COMPATÍVEL: Flask + SQLite + Python 3.11
✓ BANCO DE DADOS: Dados persistem entre deploys
✓ DORMIR: App dorme após 15 min (normal, grátis)
✓ ACORDAR: Primeira requisição demora ~30s

❌ NÃO FUNCIONA BEM:
   → SQLite em servidores com múltiplos usuários
   → SOLUÇÃO: Migrar para PostgreSQL (gratuito no Render)

⚡ ALTERNATIVAS:
   Se quiser melhor performance:
   → Railway.app ($5/mês créditos grátis)
   → PythonAnywhere (plano gratuito)
